## 写経

# ClaudeCodeのベストプラクティス

環境設定から並列セッションでのスケーリングまで、 Claude Code を最大限に活用するためのヒントとパターン。

Claude Code は agentic coding 環境です。質問に答えて待つチャットボットとは異なり、 Claude Code はファイルを読み取り、コマンドを実行し、変更を加え、あなたが見守ったり、方向を変えたり、完全に任せたりしながら、自律的に問題を解決できます。

これはあなたの作業方法を変えます。自分でコードを書いて Claude にレビューしてもらう代わりに、やりたいことを説明すると Claude がそれをどのようにこうちくするかを考え出します。 Claude は探索し、計画し、実装します。

しかし、この自立性にも学習曲線があります。 Claude は理解する必要がある特定の制約の中で動作します。

このガイドでは、　Anthropic の内部チームと、さまざまなコードベース、言語、環境で Claude Code を使用しているエンジニアの間で効果的であるということが証明されたパターンについて説明します。 agentic ループがどのように機能するかについては、 Claude Code の仕組みを参照してください。

---

ほとんどのベストプラクティスは１つの制約に基づいています。 Claude のコンテキストウィンドウはすぐにいっぱいになり、満杯になるにつれてパフォーマンスが低下します。

Claude のコンテキストウィンドウは、すべてのメッセージ、 Claude が読み取ったすべてのファイル、およびすべてのコマンド出力を含む、会話全体を保持します。ただし、これはすぐにいっぱいになる可能性があります。単一のデバッグセッションまたはコードベース探索でも、数万のトークンを生成および消費する可能性があります。

LLM のパフォーマンスはコンテキストが満杯になるにつれて低下するため、これは重要です。コンテキストウィンドウがいっぱいになると、 Claude は以前の指示を「忘れる」か、より多くの間違いを犯す可能性があります。コンテキストウィンドウは管理する最も重要なリソースです。カスタムステータスラインでコンテキスト使用量を継続的に追跡し、トークン使用量を削減するための戦略についてはトークン使用量を削減を参照してください。

## 気づき・疑問

### Q. 普段Claude Codeを使っていて、コンテキストが溢れてきた（精度が落ちた・指示を忘れた）と感じた経験はあるか？

よくある。とはいえセッションを短くするための工夫はなかなか見つけられていないのが現状。

手応えのある方法としては：
- 合意事項（チームメート・ステークホルダー・AIとの対話の中でも）をドキュメントに落とし込む
  - 理想としてはリポジトリに含めることだが、慣例や事情的に難しそうな場合は別のフォルダを用意してこまめに残していく
  - そうすると、ファイルを通してセッション間で情報を渡せる

できていないこととしては：
- 規約やsubagent, agent teamsなどでクリアな情報・環境を用意すること
  - 概念として情報は目にするが、自分のハンズオンまで落とし込めていないのが現状

> **Claudeのコメント:** ドキュメントに落とし込んでセッション間で情報を渡すというのは、記事の後半で出てくるCLAUDE.mdやスキルと本質的に同じアプローチ。すでに感覚は掴んでいるが、Claude Codeのエコシステム（スキル・サブエージェント・フック等）としてのハンズオンが追いついていない。この写経プロジェクトでセクション05（環境を設定する）に進んだときに、具体的に手を動かす機会になるはず。

### Q. 「チャットボットではなくagentic coding環境」という区別について、自分の使い方はどっち寄りか？

どちらかというと質問→回答の往復になってしまっている感覚がある。タスクを渡して自律的にやらせるところまでなかなかできていない。

質問→回答の往復で認識を合わせる形になってしまっている。エコシステムを充実させて、タスクを自律的にお願いできる形にもっていくのが理想。

また、言語化できていないタスクを減らして、任せるタスクの幅を広げたいというところもある。

> **Claudeのコメント:** チャットボット的になってしまう根本原因は、Claudeに自律的に動いてもらうための下地（環境整備）が足りないから。記事の構成自体がそれを裏付けていて、検証方法を与える（02）→環境を設定する（05）→セッション管理（07）はすべて「自律の下地作り」。このプロジェクト自体が、そのギャップを埋める取り組みになっている。

## 自分の仕事に使えそうなこと

- 合意事項のドキュメント化は継続する。そのうえで、リポジトリに入れられるものはCLAUDE.mdやスキルとして整備する方向に進化させたい
- 「言語化できていないタスク」を減らすこと自体が、この写経プロジェクトのゴールの一つ。ベストプラクティスの語彙を自分のものにすることで、Claudeに渡せるタスクの粒度が上がるはず
